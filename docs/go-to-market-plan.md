# Tuk Talk Thai Go-To-Market Plan

Date: 2026-05-25

This plan assumes a near-term web/PWA soft launch first, followed by native store preparation after feedback.

## Positioning

Tuk Talk Thai is practical Thai for real life in Thailand. It helps travelers, expats, and serious beginners speak useful Thai early, then retain it through smart review.

Core message:

> Real Thai for real life in Thailand. Speak from day one.

Avoid:

- Claims that guarantee fluency in a specific number of days.
- Copying Duolingo wording, characters, visuals, or UI patterns.
- Overpromising features that are still placeholders, such as Shop, Hearts, Premium, or Leaderboards.

## Target user

Primary:

- English-speaking travelers, expats, digital nomads, and long-stay visitors preparing for Thailand.
- Learners who want food, taxi, shopping, polite basics, help, directions, and everyday conversation.

Secondary:

- Thai-language beginners frustrated by textbook-first learning.
- Learners who tried generic flashcard decks but want a guided path.
- Partners/friends of Thai speakers who need practical phrases quickly.

Not the first target:

- Advanced Thai readers.
- Classroom teachers needing lesson management.
- Children or school deployments.
- Users expecting full speech recognition or live tutoring.

## Paid ad test plan

Do not start paid ads until:

- The domain is live.
- Privacy/terms/support pages are live.
- Sign-up, first lesson, and progress sync pass smoke tests.
- Basic analytics can measure landing visit, signup, and first lesson completion.

First Meta test:

| Item | Plan |
|---|---|
| Budget | $10-$20/day for 5-7 days |
| Objective | Website conversion or landing-page signup |
| Audience 1 | Thailand travelers, Bangkok, Phuket, Chiang Mai, Thai language |
| Audience 2 | Expats, digital nomads, language learning |
| Creative A | "Thai for food, taxis, prices, and help" |
| Creative B | "Speak useful Thai from day one" |
| Landing | Main PWA landing page |
| Success metric | Cost per account created and cost per first lesson completed |
| Stop condition | Poor first lesson completion or support issues |

## Tracking metrics

Acquisition:

- Landing page visitors.
- Traffic source.
- Signup click-through rate.
- Account creation rate.
- Cost per account if ads run.

Activation:

- Email confirmation rate.
- Placement onboarding completion.
- First lesson start.
- First lesson completion.
- Demo completion and demo-to-signup conversion.

Retention:

- Day-1 return.
- Day-3 return.
- Day-7 return.
- Streak started.
- Push opt-in.
- Cards reviewed per active user.
- Challenge completion.

Product quality:

- Support requests per 100 users.
- Auth failures.
- Progress sync failures.
- Notification subscription failures.
- Content reports.
- Mobile layout issues by device.

Monetization signals:

- Founder/supporter interest clicks.
- Pricing survey responses.
- Repeat users asking for more units or paid packs.
- Willingness to pay from user interviews.

## Launch execution notes

- Start with a small trusted group before public ads.
- Ask each tester to complete the first lesson and one Cards session.
- Interview at least five users before changing monetization.
- Use real user language in future landing copy.
- Keep the first week focused on activation and retention, not revenue.
