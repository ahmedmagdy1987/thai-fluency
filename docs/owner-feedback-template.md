# Owner Feedback Template

Copy this block once per testing session (or per issue) and fill it in. Send to
the development chat or keep in this file; anything marked "blocker" gets fixed
first.

```
== Tuk Talk Thai beta feedback ==

Date:
Device (phone/tablet/desktop model):
Browser or APK: (e.g. Chrome on Android / Safari iPhone / app-debug.apk)
Screen size if known: (e.g. 6.1", 360x800, "small phone")
Signed in or signed out:

--- The good ---
What felt exciting:

What felt smooth or surprisingly nice:

--- The bad ---
What felt boring (where did you lose interest?):

What was confusing (what did you expect instead?):

Any broken button or dead end (where, what happened):

Any text too long, cut off, or overlapping (screen + what it said):

--- Content ---
Any Thai that needs native review (word/sentence + what looks wrong):

Any English copy that overclaims or reads oddly:

--- Priority ---
Overall priority of the worst issue found:
  [ ] blocker (stops testing / breaks core flow)
  [ ] important (hurts the experience, must fix before beta invite)
  [ ] polish (nice to fix later)

Screenshots attached: yes / no
```

Tip: the fastest useful feedback is "I expected X, but Y happened, on screen Z."
