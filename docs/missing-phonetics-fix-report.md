## Why so few mechanical fixes?

Three lookup strategies were attempted (see `scripts/audit-missing-phonetics.mjs`):

1. **Exact-Thai match against another card with `ph` filled.**
   Result: 0 matches. None of the 334 missing-phonetic Thai strings appear
   anywhere else in `CARDS` with a filled phonetic. Each idiom is unique to
   the import batch.
2. **Exact-Thai match against `src/data/lookup.js`.**
   Result: 0 matches. `lookup.js` indexes the canonical single-word vocabulary
   used for auto-breakdowns; these sentence-level idioms are not in it.
3. **Tail-particle decomposition** (e.g. "X + ครับ" where the head X has a
   known phonetic).
   Result: 0 decomposable cards.

Component-wise reconstruction of the remaining cards was deliberately *not*
attempted: each phrase is a colloquial idiom or contains slang particles,
reduplication (ๆ), or compound nouns where tone sandhi, vowel length, and
juncture can shift the surface pronunciation. Per the task brief — and the
project rule "do not guess Thai" — guessing here would be a net negative for
a Thai-learning app: a wrong tone in a learner's deck is worse than no
phonetic at all. The lesson UI handles empty `ph` gracefully via a
"phonetic coming soon" placeholder.

## How a native-speaker pass should run

1. Walk the table below stage-by-stage (the report is sorted that way).
2. For each card, fill the **New Phonetic** column using the app's existing
   romanization convention: doubled vowels for length where applicable,
   tone marks `à á â ǎ` on the first vowel, hyphens within compound nouns,
   spaces between distinct words. See `src/data/cards.js` for canonical
   examples.
3. Run `node scripts/audit-missing-phonetics.mjs` after each batch to track
   the remaining count.
4. Once a batch of phonetics has been verified, add them as
   `STEP2_OVERRIDES` entries in `src/data/cards-step2.js` keyed by card id,
   e.g.:
   ```js
   4772: { ph: "ôh hǒh" },  // โอ้โห
   ```
   This keeps the auto-generated imported files clean of hand-edits and
   makes the changes auditable in version control.
